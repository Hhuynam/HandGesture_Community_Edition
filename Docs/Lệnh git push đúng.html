Lệnh git push origin main đúng nếu nhánh chính trong repository của bạn tên là main và bạn muốn đẩy (push) code từ nhánh main lên remote repository (origin).

Tuy nhiên, nếu repository của bạn đang dùng nhánh master thay vì main, thì lệnh đúng sẽ là:

bash
git push origin master

Để kiểm tra nhánh hiện tại, bạn có thể dùng:

bash
git branch

Nếu bạn chưa thiết lập upstream cho nhánh main, có thể cần thêm -u:

bash
git push -u origin main

Lệnh này sẽ thiết lập nhánh main trên remote làm mặc định để push về sau chỉ cần git push.

Lỗi non-fast-forward xảy ra khi nhánh main trên GitHub đã có thay đổi mới hơn so với nhánh main trên máy của bạn. Có hai cách để xử lý:

Cách 1: Kéo (pull) thay đổi từ remote trước rồi push lại

Thử:

Lệnh git pull origin main --rebase sẽ kéo code mới từ GitHub về và áp dụng các commit của bạn lên trên cùng, tránh tạo merge commit không cần thiết. Sau đó, git push origin main sẽ hoạt động bình thường.

model.save("my_model.h5")

  • Không nén như .keras, nên có thể chiếm nhiều dung lượng hơn.

🚀 3. Lưu model dưới dạng tách biệt (Weights + Architecture)

Nếu bạn muốn lưu riêng kiến trúc và trọng số:

model.save_weights("my_model.weights.h5")

🔹 Lưu chỉ kiến trúc model (.json):

python
model_json = model.to_json()
with open("my_model.json", "w") as json_file:
    json_file.write(model_json)

🔹 Khôi phục model từ .json + .weights.h5:

python
from tensorflow import keras
import json

# Load model từ JSON
with open("my_model.json", "r") as json_file:
    loaded_model_json = json_file.read()
loaded_model = keras.models.model_from_json(loaded_model_json)

# Load trọng số
loaded_model.load_weights("my_model.weights.h5")

👉 Ưu điểm: Dễ quản lý riêng kiến trúc và trọng số.
👉 Nhược điểm: Phải lưu cả 2 file .json và .weights.h5.

❓ Nên chọn cách nào?

  • Dùng .keras nếu bạn xài TensorFlow 2.11+ và muốn nén model.
  • Dùng .h5 nếu muốn tương thích với TensorFlow cũ hơn hoặc dễ chia sẻ.
  • Dùng JSON + Weights nếu bạn cần tách riêng kiến trúc và trọng số.
